# Kit Package (`packages/kit`)

Hosted receipt-validation SaaS at [kit.openiap.dev](https://kit.openiap.dev) — React 19 SPA + Hono on Bun + Convex backend, all shipped as one Fly.io binary.

## Critical: Different from Other Packages

`packages/kit` is the only package in this monorepo that is **a deployable application, not a publishable library**. Treat it differently:

| Aspect              | apple/google/gql/docs       | kit                                                    |
| ------------------- | --------------------------- | ------------------------------------------------------ |
| Output              | Library / static site       | Running SaaS (Fly.io machine)                          |
| Type SSOT           | `packages/gql` (GraphQL IR) | Independent Convex schema                              |
| Deploy trigger      | Tagged release              | `main` push (paths-filtered to `packages/kit/**`)      |
| GQL type-sync chain | Yes                         | **No** — kit does not consume `@hyodotdev/openiap-gql` |
| `private: true`     | Mixed                       | Yes — never publish to npm                             |
| User-facing brand   | `openiap-*`                 | `IAPKit` (managed by OpenIAP)                          |

## Internal Layout

```text
packages/kit/
├── src/         # React 19 SPA (English only — see below)
├── server/      # Hono + Bun (/api/v1/* receipt verification + SPA fallback)
├── convex/      # Convex functions (auth, orgs, projects, receipts, files)
├── public/      # Static assets (favicons, OG image, doc screenshots)
├── scripts/     # Doc captures, deploy scripts, Apple cert refresh
├── Dockerfile   # Multi-stage → single-binary Fly.io image
├── fly.toml     # Fly app: `openiap-kit` (do NOT rename — bound to prod identity)
├── README.md    # User-facing setup, ops, deploy
└── CONVENTION.md  # Internal coding rules
```

## Language Policy

The dashboard ships **English-only** by design (same as RevenueCat / Stripe / Linear / Vercel / Convex / Sentry / Mixpanel dashboards). The original `openiap-kit` repo had 14 locale files; on integration into the monorepo we removed `react-i18next`, `i18next`, `src/locales/`, `src/lib/i18n.ts`, `src/components/LanguageSwitcher.tsx`, and `scripts/check-i18n.mjs`, and inlined every `t(...)` call to its English literal.

Do not reintroduce i18n upstream without discussion. The simplification was intentional, not an oversight — see `packages/kit/CONVENTION.md` ("Language" section).

## Convex Backend

Domain folders under `convex/` follow CQRS:

```text
convex/<domain>/
├── query.ts      # Public reads
├── mutation.ts   # Public writes
├── action.ts     # External APIs / cross-function orchestration
└── internal.ts   # Internal-only (called from actions)
```

`convex/_generated/*` is regenerated by `bunx convex dev` — never edit. When working on Convex code, **read `convex/_generated/ai/guidelines.md` first**.

## Pre-commit Gate (Paths-Aware, CI-Equivalent)

The monorepo-root husky hook (`.husky/pre-commit`) runs the **full CI-equivalent gate** when staged changes touch `packages/kit/**`: `bun install --frozen-lockfile`, lint (tsc + eslint), prettier check, vitest, and `smoke:server` (Bun compile + boot probe). Mirrors the `verify` job in `deploy-kit.yml` exactly so issues that only surface on CI's fresh install are caught locally. ~15-20s on warm checkouts, ~30-60s after a clean install.

If the hook fails, fix the underlying issue and re-stage; never bypass with `--no-verify`.

## Secrets

GitHub Actions secrets (set on `hyodotdev/openiap`):

| Secret                    | Real secret? | Purpose                                                  |
| ------------------------- | ------------ | -------------------------------------------------------- |
| `KIT_FLY_API_TOKEN`       | ✅ yes       | `flyctl deploy` auth — keep private                      |
| `KIT_CONVEX_DEPLOY_KEY`   | ✅ yes       | Convex function deploy (optional — step skips if absent) |
| `VITE_KIT_CONVEX_URL`     | ⚠️ public    | Build arg for SPA — visible in deployed JS bundle        |
| `VITE_KIT_SENTRY_DSN`     | ⚠️ public    | Build arg for SPA (optional)                             |
| `VITE_KIT_MIXPANEL_TOKEN` | ⚠️ public    | BuildKit secret for SPA (optional, analytics opt-in)     |

`VITE_KIT_*` values are inlined into the SPA bundle at `bun run build`; treat them as public configuration, not secrets.

Auth-provider secrets (Resend API key, GitHub OAuth client + secret) and Apple root certificates live in the **Convex dashboard** (Settings → Environment Variables), not in Fly secrets or `.env.example`.

## Origin Guard for Analytics

Forks self-hosting from `packages/kit` must NOT pipe their users' Mixpanel events or Sentry errors into the OpenIAP-managed projects. `src/lib/mixpanel.ts` and `src/lib/sentry.ts` both check `window.location.hostname` against an `ALLOWED_HOSTS` whitelist (`kit.openiap.dev`, `localhost`, `127.0.0.1`) and skip init on unrecognized hosts. Do not remove this guard.

## VS Code Launch

Monorepo `.vscode/launch.json` has a single kit entry: **🧰 Kit: Dev (Vite + Hono + Convex)**. It runs `bun install` if `node_modules` is missing at workspace root, then launches Convex + Hono + Vite in parallel and tears them down together via `kill 0`.

## When You Touch Kit

- Stay paths-aware. The deploy workflow only fires on `packages/kit/**` changes.
- Add new env vars to `.env.example` first (template), then `.env.local` (dev) and `.env.production` (manual prod fallback). For `VITE_KIT_*` vars, also update the Docker build-time injection in `Dockerfile`, the `Deploy` step in `deploy-kit.yml`, and the GitHub secrets. Use BuildKit secrets for TOKEN-named public SPA values to avoid Docker secret-name warnings.
- For server-runtime-only secrets (Stripe / Resend / GitHub OAuth), use the Convex dashboard, not these files.
- Keep dashboard text English-only. Inline string literals; do not reintroduce i18next.
